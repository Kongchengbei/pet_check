# 宠标通 · 阿里云部署指南

## 前提：你需要准备的

| 事项 | 说明 |
|------|------|
| 阿里云 ECS 实例 | 建议 Ubuntu 22.04，2核4G 内存起步 |
| 安全组规则 | 开放 80（HTTP）、443（HTTPS）、22（SSH）端口 |
| SiliconFlow API Key | LLM Token，在 https://siliconflow.cn 获取 |
| 百度 AIStudio Token | OCR Token，在百度 AIStudio 控制台获取 |
| 公网 IP | ECS 分配的公网地址 |

---

## 第一步：连接到 ECS

在你 Windows 电脑上打开 PowerShell 或 CMD：

```bash
ssh root@你的ECS公网IP
```

---

## 第二步：初始化服务器（一次性）

SSH 登录后，运行 deploy-files/deploy/ecs-setup.sh 的内容：

```bash
# 更新系统
apt-get update && apt-get upgrade -y

# 安装 Docker（使用阿里云镜像加速）
apt-get install -y ca-certificates curl
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null
apt-get update
apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 启动 Docker
systemctl enable docker
systemctl start docker

# 创建应用目录
mkdir -p /opt/pet-feed-backend/temp_images
mkdir -p /opt/pet-feed-backend/tasks
```

---

## 第三步：上传后端代码

在你的 Windows 电脑上，把整个后端项目传到服务器。

首先在你的电脑上准备好要上传的文件夹。你需要把**源码里的** `pet-feed-backend` 文件夹（包含 app.py、config.py 等所有 .py 文件 + Dockerfile + requirements.txt + Utils/），加上我给你的 `deploy-files/` 里的文件，合并到一起。

用 PowerShell 操作：

```powershell
# 假设你的项目在 F:\源码111Pet_Feed_Check (2)
cd "F:\源码111Pet_Feed_Check (2)\Pet_Feed_Check (2)\111Pet_Feed_Check\Pet_Feed_Check\pet-feed-backend"

# 把新加的部署文件也放进去
copy "F:\源码111Pet_Feed_Check (2)\newwork\deploy-files\docker-compose.yml" .
copy "F:\源码111Pet_Feed_Check (2)\newwork\deploy-files\.env.example" .

# 用 scp 上传整个文件夹到服务器
scp -r * root@你的ECS公网IP:/opt/pet-feed-backend/
```

---

## 第四步：配置环境变量

SSH 回到服务器：

```bash
cd /opt/pet-feed-backend

# 从模板创建 .env 文件
cp .env.example .env

# 编辑 .env，填入真实的 Token
nano .env
```

`.env` 文件内容如下（把你的真实 Token 填进去）：

```
OCR_TOKEN=你的百度AIStudio_Token
LLM_TOKEN=sk-你的SiliconFlow_API_Key
PORT=80
DEBUG=false
```

保存：`Ctrl+O` → 回车 → `Ctrl+X`

---

## 第五步：启动服务

```bash
cd /opt/pet-feed-backend

# 构建镜像并启动
docker compose up -d --build

# 查看是否启动成功
docker compose ps

# 查看日志（确认没有报错）
docker compose logs -f
```

等待大约 2-3 分钟（首次构建需要下载依赖）。看到类似 `Starting gunicorn...` `Listening at: http://0.0.0.0:80` 就成功了。

---

## 第六步：验证

在浏览器访问：

```
http://你的ECS公网IP/
```

应该看到 JSON 返回：

```json
{
  "service": "宠物食品标签审核服务",
  "version": "local-debug",
  "endpoints": {...}
}
```

再测试健康检查：

```
http://你的ECS公网IP/api/health
```

返回 `{"status": "ok"}` 即可。

## 第七步：配置 iOS App

在 iPhone/iPad 上打开宠标通 App → 右上角齿轮进入设置：

| 配置项 | 填什么 |
|--------|--------|
| 后端地址 | `http://你的ECS公网IP` |
| LLM Token | 同 .env 里的 LLM_TOKEN |

保存后即可使用。

## （可选）绑定域名 + HTTPS

```bash
# 安装 Nginx
apt-get install -y nginx

# 复制 nginx 配置
cp /opt/pet-feed-backend/deploy/nginx.conf /etc/nginx/sites-available/pet-feed-backend
ln -s /etc/nginx/sites-available/pet-feed-backend /etc/nginx/sites-enabled/

# 修改配置文件中的域名
nano /etc/nginx/sites-available/pet-feed-backend
# 把 your-domain.com 改成你的真实域名

# 重启 Nginx
systemctl restart nginx

# 安装 SSL 证书（Let's Encrypt 免费）
apt-get install -y certbot python3-certbot-nginx
certbot --nginx -d 你的域名
```

## 故障排查

**Q: 构建镜像很慢**  
A: Dockerfile 已配置阿里云 pip 镜像，首次构建约 3-5 分钟。耐心等待。

**Q: 启动后 curl localhost:80 不通**  
A: 检查安全组是否开放了 80 端口（阿里云控制台 → ECS → 安全组 → 入方向规则）。

**Q: OCR 调用报 "Token 无效"**  
A: 百度 AIStudio Token 有时效性，过期后需重新获取。去 AIStudio 控制台刷新。

**Q: LLM 调用超时**  
A: `docker-compose.yml` 中 gunicorn timeout 已设为 180 秒。如果图片很大，模型推理本身就需要 1-2 分钟，这是正常的。
