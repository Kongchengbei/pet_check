# 宠物食品标签审核系统 OCR 适配优化实验报告

## 1. 报告概述

本报告面向“宠物食品标签智能合规审核系统”的 OCR 环节，围绕宠物食品标签场景中的识别难点，提出一套可落地的 OCR 适配优化方案，并给出实验流程、样本设计、评测指标与结果记录模板。

结合当前系统实现方式，项目中的 OCR 并非本地自训练模型，而是通过后端接口调用第三方 OCR 服务完成识别。因此，本阶段工作的重点不表述为“底层模型微调完成”，而表述为“面向宠物食品标签场景的 OCR 适配优化设计与实验验证”。该表述与当前系统实现一致，也便于后续继续补充真实实验数据。

本报告可作为课程设计、项目中期检查、系统实验记录或结题材料中的 OCR 专项小节使用。

## 2. 项目背景

本系统的目标是对宠物食品标签图片进行智能审核，自动识别包装上的产品名称、原料组成、添加剂、成分保证值、生产许可证号、执行标准、保质期等信息，并结合法规资料与大语言模型输出合规性判断结果。

在实际使用过程中，OCR 环节直接影响下游审核质量。若 OCR 存在漏字、错字、顺序紊乱或字段缺失等问题，将导致后续法规检索与合规判定结果出现偏差。因此，有必要针对宠物食品标签场景中的复杂版面、包装反光、小字号密集排版和多图联合输入等问题，开展 OCR 适配优化研究。

## 3. 当前系统中的 OCR 处理链路

### 3.1 系统实现现状

当前项目后端采用 Flask 框架构建服务，整体审核链路如下：

1. 小程序前端上传单张或多张宠物食品标签图片。
2. 后端接收图片后，将多图进行统一处理。
3. OCR 模块对图片进行文字识别，输出 Markdown 文本。
4. 文本检索模块从法规知识库中召回相关条文。
5. 大语言模型基于 OCR 文本与法规资料生成合规审核结果。

### 3.2 OCR 模块现状说明

当前项目中的 OCR 位于后端 `pet-feed-backend/Paddle_OCR.py`，采用第三方 OCR 接口完成版面解析和文字识别。多张图片的处理逻辑位于 `pet-feed-backend/Function_Management.py`，默认流程为“多图先拼接，再进行统一 OCR”。

因此，当前阶段的优化重点包括：

- 图像输入流程优化
- 多图识别策略优化
- 识别结果后处理优化
- 关键字段纠错与结构化提取优化

而不直接涉及底层 OCR 模型权重训练。

## 4. 研究目标

本阶段实验的目标如下：

1. 提升宠物食品标签图片在复杂拍摄条件下的文本识别可用性。
2. 提升关键字段的识别完整率和准确率。
3. 降低多图联合识别场景中的版面混乱问题。
4. 为后续法规审核模块提供更稳定、更结构化的文本输入。

## 5. 宠物食品标签场景中的 OCR 难点

结合实际业务场景，宠物食品标签 OCR 主要面临以下问题：

- 包装材质反光，导致局部文字发白、发虚或缺失。
- 标签区域存在曲面变形，拍摄后文字发生透视畸变。
- 配料表和成分保证值区域字体较小，容易出现识别错误。
- 标签中包含表格、分栏和段落混排，导致识别顺序错乱。
- 多张图片拼接后版面结构被拉长，可能影响布局检测与文本顺序恢复。
- 关键字段中数字、单位和符号较多，如 `%`、`≥`、`≤`、许可证编号等，易发生细粒度误识别。

## 6. 优化思路

本研究将 OCR 优化分为三个层面。

### 6.1 图像处理优化

面向输入图片进行预处理，提升图像可读性，主要包括：

- 分辨率统一
- 图片方向校正
- 对比度增强
- 锐化处理
- 无效边缘裁切

### 6.2 多图识别流程优化

当前系统默认采用“多图拼接后统一 OCR”的方式。为验证不同流程的效果，设计如下对比方案：

- A 组：保持现有流程，多图先拼接再 OCR。
- B 组：逐张图片独立 OCR，再按顺序合并文本。
- C 组：逐张图片独立 OCR，并在合并前进行关键字段纠错。

### 6.3 领域后处理优化

结合宠物食品标签中的高频字段，对 OCR 结果进行规则增强与词典纠错，重点字段包括：

- 产品名称
- 原料组成
- 添加剂
- 粗蛋白
- 粗脂肪
- 水分
- 生产许可证号
- 执行标准
- 保质期
- 净含量

后处理方法包括：

- 基于关键词的字段定位
- 数值与单位正则纠错
- 许可证号格式修正
- 多余换行与乱码清洗
- 标签顺序重组

## 7. 实验设计

### 7.1 数据集构成

考虑到项目周期和实现成本，本阶段构建一个小规模场景验证数据集，共采集 30 张真实宠物食品标签图片，分为三类：

| 图像类别 | 数量 | 说明 |
| --- | ---: | --- |
| 清晰正拍图像 | 10 | 光照正常、角度较正、文字清晰 |
| 反光/弯曲/倾斜图像 | 10 | 存在包装反光、透视畸变或轻度模糊 |
| 小字密集/复杂表格图像 | 10 | 成分表、配料表或参数区文字密集 |
| 合计 | 30 | 用于 OCR 适配优化实验 |

### 7.2 标注方案

为降低标注成本，同时保证实验可比性，本阶段不对全部文本做逐字人工转写，而是对关键字段进行结构化标注。每张图片标注 8 类关键字段：

| 字段名称 | 是否标注 |
| --- | --- |
| 产品名称 | 是 |
| 原料组成 | 是 |
| 添加剂 | 是 |
| 粗蛋白 | 是 |
| 粗脂肪 | 是 |
| 水分 | 是 |
| 生产许可证号 | 是 |
| 保质期 | 是 |

`30 × 8 = 240 个字段实例`

| 组别 | 方法说明 | 目标 |

## 8. 评测指标

本实验从字段级和标签级两个角度进行评测。

### 8.1 字段召回率

字段召回率用于衡量应识别出的字段中，系统成功识别出的比例。

公式如下：

`字段召回率 = 成功识别的目标字段数 / 标注字段总数`

### 8.2 字段准确率

字段准确率用于衡量系统识别出的字段中，内容正确的比例。

公式如下：

`字段准确率 = 识别正确的字段数 / 系统输出的字段数`

### 8.3 标签可用率

标签可用率用于衡量 OCR 输出是否能够直接支持后续审核。

当一张图片中关键字段大部分可读，且内容顺序基本正确时，可判定为“可用”。

公式如下：

`标签可用率 = 可直接用于审核的图片数 / 总图片数`

### 8.4 平均处理时间

平均处理时间用于衡量不同 OCR 流程的实际运行效率。

公式如下：

`平均处理时间 = 总处理耗时 / 图片总数`

## 10. 结果分析模板

### 10.1 总体结果分析

从总体实验结果看，B 组和 C 组在字段召回率、字段准确率与标签可用率方面预计优于 A 组。其原因在于逐张图片独立 OCR 能够保留原始版面信息，避免多图拼接后造成布局拉伸和文本顺序扰动。同时，结合宠物食品标签领域词典与规则纠错后，可进一步减少关键字段在数字、单位和专有术语上的识别偏差。

在平均处理时间方面，逐张 OCR 流程可能略高于拼接后统一识别流程，但若识别质量提升明显，该时间代价是可以接受的。

### 10.2 分类场景分析

1. 清晰正拍图像  
该类图像通常能够获得较高识别率，主要作为基线参考。若该类样本中仍出现字段错误，则说明 OCR 对细粒度数字或版面结构存在固有缺陷。

2. 反光/弯曲/倾斜图像  
该类图像受拍摄条件影响明显，容易出现漏字、跳行和段落截断。若逐张 OCR 较基线方法有明显提升，则说明多图拼接对复杂场景的负面影响较大。

3. 小字密集/复杂表格图像  
该类图像通常是 OCR 难点。若 C 组在该类场景中提升最明显，则说明领域后处理对于关键字段提取具有较高价值。

### 10.3 典型失败案例分析模板

可选取 3 至 5 个失败样本进行分析，建议从以下角度描述：

- 原始图像特点
- A 组识别结果存在的问题
- B 组或 C 组的改进情况
- 失败原因分析
- 后续改进方向

示例描述：

> 样本 014 为反光较强的包装正面图像。A 组方法在 OCR 结果中漏识别了“保质期”字段，同时将“粗脂肪 ≥ 12.0%”错误识别为“粗脂肪 ≥ 1 2.0”。B 组逐张识别后，字段完整率有所提升，但数值间距问题仍然存在。C 组在加入规则纠错后，能够正确恢复数值格式，说明领域后处理对含单位和符号的字段具有明显增益。

## 11. 结论模板

本阶段围绕宠物食品标签审核系统中的 OCR 环节，完成了面向业务场景的适配优化设计，并建立了小规模实验评测方案。实验从图像处理、多图识别流程和领域后处理三个层面提出优化思路，构建了覆盖清晰正拍、反光弯曲和小字密集场景的 30 张标签图片评测集，并以 8 类关键字段作为结构化评测对象，形成 240 个字段级样本。

通过设置“多图拼接统一 OCR”“逐张 OCR”“逐张 OCR + 领域纠错”三组对比方案，可以系统评估 OCR 适配优化对识别完整率、准确率和标签可用性的影响。该实验设计能够为后续系统改进提供明确依据，也为进一步开展本地 OCR 模型训练或专项微调研究奠定基础。

## 12. 后续工作

后续工作主要包括以下几个方向：

1. 补充更多真实标签图片，扩大样本规模。

3. 将逐张 OCR 与字段纠错流程正式集成到现有后端服务中。
4. 构建更细粒度的字符级标注集，开展字符错误率评测。
5. 在条件允许时，切换至可训练 OCR 模型，开展真正的领域微调实验。

## 13. 附录：可直接引用的简版摘要

本研究面向宠物食品标签智能合规审核系统中的 OCR 环节，针对包装反光、曲面变形、小字号密集排版和多图联合识别等问题，提出 OCR 适配优化方案。考虑到当前系统采用第三方 OCR 接口，本文未直接进行底层模型重训练，而是从图像处理、多图识别流程和领域后处理三个层面开展优化设计。实验构建了包含 30 张真实宠物食品标签图片的小规模评测集，并围绕产品名称、原料组成、添加剂、粗蛋白、粗脂肪、水分、生产许可证号和保质期共 8 类关键字段进行结构化评测。通过设置多图拼接 OCR、逐张 OCR 和逐张 OCR 结合领域纠错三组对比方案，可从字段召回率、字段准确率、标签可用率和平均处理时间等指标评估优化效果。研究结果将为系统后续 OCR 能力提升和业务链路稳定性优化提供依据。
